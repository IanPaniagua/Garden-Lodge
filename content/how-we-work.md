## The loop

1. **Set a goal** for the week or fortnight, and break it into clear tasks so everyone knows what success looks like.
2. **Learn how it's done** — we research, ask experienced gardeners, and collect practical advice before starting. Knowledge first, then action.
3. **Apply it** in the garden, hands-on, as a team — plant, tend and test approaches together so learning spreads quickly.
4. **Leave it cleaner and better** than we found it — a little more each cycle.

## We run cycles like small agile sprints

Each cycle is a sprint with a single clear goal and a visible result. We keep work
transparent so anyone can see what's done, in progress, and next. Ideas go into a
shared backlog and are pulled into the next sprint — that way no useful suggestion
gets lost.

- **Sprint board** — a simple, visible board showing Done, In progress and Next, updated each sprint by the core group.
- **Backlog & ideas** — every idea is recorded in a shared backlog so it can be prioritised and scheduled into upcoming sprints.

> Link the Sprint board and Backlog buttons to the public Notion views rather than
> rebuilding them, so there is a single source of truth to maintain.
